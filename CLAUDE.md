# ai-linkedin-job-searcher

Nx monorepo. Currently one app: `apps/app-be`, a Python/FastAPI backend for AI-assisted LinkedIn job search. See [`apps/app-be/README.md`](apps/app-be/README.md) (setup/run/test) and [`apps/app-be/src/app_be/README.md`](apps/app-be/src/app_be/README.md) (architecture) for full detail. Deeper, topic-scoped conventions live in `.claude/rules/` and load automatically when relevant.

## Approach

- Read existing files before writing. Don't re-read unless changed.
- Thorough in reasoning, concise in output.
- Skip files over 100KB unless required.
- No sycophantic openers or closing fluff.
- No emojis or em-dashes.
- Do not guess APIs, versions, flags, commit SHAs, or package names. Verify by reading code or docs before asserting.

## Coding Standards

- Python, `uv`-managed, pinned to 3.11.15 via `apps/app-be/.python-version`.
- Linted/formatted with `ruff` (line-length 88, rule sets E/F/UP/B/SIM/I — see `apps/app-be/pyproject.toml`).
- Layering rule: `main.py` → `services/` → `api_clients/`. Full detail in `.claude/rules/python-architecture.md` and `apps/app-be/src/app_be/README.md`.
- Env vars: only `apps/app-be/src/app_be/config.py` reads them. See `.claude/rules/secrets-and-env.md`.
- Tests: pytest. See `.claude/rules/python-testing.md`.

## Commands

```bash
npx nx run app-be:serve     # FastAPI + uvicorn, hot reload, http://localhost:8000
npx nx run app-be:test      # uv run pytest tests/ (coverage + junit reports)
npx nx run app-be:lint      # ruff check
npx nx run app-be:format    # ruff format
npx nx run app-be:build     # wheel + sdist -> apps/app-be/dist/
npx nx run app-be:install   # uv sync
npx nx run app-be:add / update / remove / lock

# Nx-recognized shorthand:
npx nx test app-be
npx nx lint app-be
npx nx build app-be

# direct uv (from apps/app-be/, bypasses Nx caching):
uv run pytest tests/
uv run ruff check .
uv run ruff format --check .
```
